<Update label="2026-03-20" description="Mobile visualiser & sprint 1 maps" tags={["Feature", "Update", "Fix"]}>
  ## Mobile visualiser and sprint 1 maps

  The visualiser now works on mobile, sprint 1 maps are live, and match details show more information.

  ### New features

  - **Mobile visualiser** — The [visualiser](/getting-started/cli#cambc-watch) is now fully responsive. You can view replays, inspect tiles, and control playback on phones and tablets.
  - **Per-game winners** — Match detail pages and the visualiser now show which side won each individual game in a match, not just the overall result.

  ### Updates

  - **Sprint 1 maps** — A new set of official maps is available for local play and ladder matches.
  - **Bot load error details** — When a bot fails to load during a match, the match detail page now shows what went wrong, making it easier to debug upload and import issues.
  - **Visualiser light theme** — The visualiser now supports a light colour scheme that matches the platform's light mode.

  ### Bug fixes

  - Fixed a crash that could occur during engine execution.
  - Fixed team deletion not working on the platform.
  - Fixed bot initialisation failures causing matches to hang instead of reporting an error.
</Update>

<Update label="2026-03-19" description="Theme toggle & sandbox hardening" tags={["Feature", "Update", "Fix"]}>
  ## Theme toggle and sandbox hardening

  A new light/dark theme option, tighter bot sandboxing, and several visualiser fixes.

  ### New features

  - **Light/dark theme toggle** — You can now switch between light and dark mode from the sidebar or [Settings](https://game.battlecode.cam) page. Your preference is saved across sessions.

  ### Updates

  - **Stronger bot sandbox** — The match execution environment is now more secure. Bots can no longer spawn threads, execute shell commands, or access process memory. These changes don't affect normal bot code — only adversarial exploits are blocked.
  - **Bot source encryption** — Bot source files are encrypted on disk during remote matches, adding an extra layer of protection for your code.

  ### Bug fixes

  - Fixed cooldown counters in the [visualiser](/getting-started/cli#cambc-watch) not ticking down between rounds.
  - Fixed game result indicators in the match sidebar sometimes showing the wrong winner.
  - Fixed a bug where launchers couldn't fire under certain conditions.
  - Fixed watchdog timer reliability for time limit enforcement.
</Update>

<Update label="2026-03-18" description="API & rating improvements" tags={["Feature", "Update", "Fix"]}>
  ## API and rating improvements

  New API methods, better rating visibility, and a handful of bug fixes.

  ### New features

  - **`resign()` method** — Your bot can now forfeit a game by calling [`resign()`](/api/controller#healing--destruction). This immediately destroys your core and ends the game as a loss. Useful for conceding hopeless positions early.
  - **Build and spawn return entity IDs** — All [`build_*`](/api/controller#building) methods and [`spawn_builder()`](/api/controller#core) now return the new entity's `int` ID instead of `None`. You can immediately query or track the entity you just created.
  - **Post-game match summary** — Running [`cambc run`](/getting-started/cli#cambc-run) now prints a detailed summary after each game, including the winner, win condition, turn count, resources mined, and unit/building counts for both sides.

  ### Updates

  - **Per-team rating deltas** — Match results now show the rating change for each team individually. Previously, only one side's delta was displayed and the other was inferred.
  - **Rating baseline adjusted** — Starting ratings have been shifted from 1500 to 1200. Existing ratings were adjusted proportionally, so relative rankings are unchanged.
  - **Rating stability floor** — A minimum rating deviation prevents ratings from becoming too sticky after many games, keeping the ladder responsive to improvement.
  - **CLI v1.1.1** — Includes all changes above plus internal stability improvements. Update with `pip install --upgrade cambc`.

  ### Bug fixes

  - Fixed a visualiser animation rendering issue.
  - Fixed a stale controller bug that could cause incorrect time limit violations.
  - Fixed bot output ordering so error messages appear after the correct round.
  - Fixed a sandbox issue where bots importing certain multithreaded libraries could crash.
  - Fixed macOS resource fork files (`._ files`) polluting bot uploads.
  - Removed the virtual memory cap that was causing crashes for some bots — physical memory is still bounded at 1 GB.
</Update>

<Update label="2026-03-17" description="Launch week" tags={["Feature", "Update", "Fix"]}>
  ## Launch week

  Cambridge Battlecode is live! Here's everything that shipped this week.

  ### New features

  - **Map editor** — Create custom `.map26` maps in the browser with symmetry modes (rotational, horizontal, vertical), brush tools, undo (`Ctrl+Z`), and protobuf import/export. Map sizes are restricted to 20–50 tiles. Open it with [`cambc map-editor`](/getting-started/cli#cambc-map-editor) or on the [platform](https://game.battlecode.cam).
  - **Visualiser overhaul** — A completely rebuilt match visualiser with tile hover/select inspection, vision and attack range outlines on hover, indicator lines and dots, bot-step mode for stepping through individual unit turns, time-series resource graphs, and playback speed controls up to 64x.
  - **Marker inspector** — The visualiser sidebar now shows marker values in decimal, hex, and binary, making it easier to debug bot communication.
  - **`cambc watch` with bundled visualiser** — Running [`cambc watch`](/getting-started/cli#cambc-watch) on a local replay now starts a local server with the full visualiser built in. No internet or authentication required.
  - **WASD panning and Q/E zoom** — Navigate the visualiser with keyboard controls.
  - **Unrated matches** — Challenge any team to a match that doesn't affect ratings. Run [`cambc unrated`](/getting-started/cli#cambc-unrated) from the CLI. Unrated matches are prioritised over ladder matches for faster results.
  - **Head-to-head match filtering** — On the matches page, select two teams to filter for their direct matchups. Win/loss rows are now colour-coded for your team.
  - **Featured games** — The home page now highlights matches from top-ranked teams instead of closest-rated matches.
  - **Top teams chart** — A multi-line rating chart on the home page shows the top 10 teams over time, with your team highlighted.

  ### Updates

  - **CLI v1.0.5** — Multiple CLI releases this week with improved Windows support, updated starter bot, and bundled maps. Install the latest with `pip install --upgrade cambc`.
  - **Game balance** — Road cost scaling corrected to 0.5% (was incorrectly listed as 1%), barrier scaling contribution added, and resource distribution algorithm overhauled.
  - **Matches are 5 games** — Each match plays exactly 5 games, not best-of-5.
  - **2ms CPU time limit** — The per-round time limit is set at 2ms on AWS Graviton3 hardware. Use [`cambc test-run`](/getting-started/cli#cambc-test-run) to benchmark your bot remotely.
  - **1 GB memory limit** — Each bot is limited to 1 GB of memory per match.
  - **Python 3.12 and 3.13 only** — Bots must target Python 3.12 or 3.13. Python 3.14 is not supported.
  - **Student status fix** — Your student/non-student status now displays correctly in settings and is editable.
  - **Matches page performance** — Active matches are capped at 50 rows with a count of additional queued matches, reducing page load times during peak scheduling.

  ### Bug fixes

  - Fixed Windows encoding issues with `cambc.toml` — files written with non-UTF-8 encoding are now auto-repaired on read.
  - Fixed maps not being bundled in the Windows CLI wheel.
  - Fixed a crash on Python 3.12 related to sub-interpreter shutdown.
  - Fixed time limit enforcement reliability for bot execution.
  - Fixed foundry resource distribution bug.
  - Fixed markers being placeable on walls — they no longer are.
  - Fixed map editor allowing overlapping core placement areas.
  - Fixed visualiser texture loading when running `cambc watch` locally.
</Update>
